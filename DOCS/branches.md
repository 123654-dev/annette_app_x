# Branches

* master: NICHT anfassen, außer für Pull Requests nach Änderungen auf anderen Branches

* organisation: Ausschließlich für Dokumentation und Verwaltung der Ordnerstruktur.

Für alles andere sollte grundsätzlich für jede Änderung ein eigener Branch erstellt werden, der dann nach Abschluss des Features in den master-Branch gemerged wird. So entstehen weniger Merge Conflicts als letztes Mal...
